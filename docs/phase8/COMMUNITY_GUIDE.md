# Phase 8 Community Guide

Date: July 16, 2026

## Intended Audience

- autonomy researchers
- systems engineers
- validation contributors
- documentation contributors

## How To Engage

- read `README.md` for architecture and workflow overview
- read `CONTRIBUTING.md` before proposing changes
- preserve prior evidence and limitations
- add scripts and machine-readable outputs for new experiments

## Good First Contributions

- benchmark schema improvements
- documentation clarification
- reproducibility helpers
- experiment metadata tooling
- non-flight simulation scenarios

## Collaboration Principles

- evidence first
- reproducibility over anecdote
- safety wording must stay explicit
- no fabricated validation claims

## Reporting Issues

Useful issue reports should include:

- environment
- commands run
- expected result
- observed result
- relevant artifact paths
